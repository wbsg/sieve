1. About

Sieve data quality assessment and data fusion framework provides a library of quality assessment scores and fusion functions, and an XML-based input specification language to combine them. Howevere, manually defining a fusion policy, i.e. which fusion function and with what parameters to use for which data attribute, requires certain domain knowledge and understanding of the input data, is time-consuming, and does not guarantee an optimal result.

Therefore, we introduce the Fusion Policy Learner, an extension of Sieve that allows automatically selecting an optimal fusion function based on the gold standard. The user is still involved in the process as the list of possible fusion functions per property needs to be manually specified.

The learning algorithm implemented in the Fusion Policy Learner selects the fusion function that minimizes the error with respect to the gold standard. The list of fusion functions and the respective quality assessment scores for each property are defined by the user in the input specification, which is written using the extended version of the XML-based Sieve specification language.

The algorithm first detects whether the values to fuse are numeric or nominal; examples of nominal attributes are strings and URIs. Then, one of the following strategies are applied: (1) a fusion function that minimizes the mean absolute error with respect to the gold standard is selected, or (2) given a maximum error threshold (e.g. 5%), the fusion function that maximizes the number of values that deviates from a respective gold standard value no more than by a threshold, is selected.

2. Input specification

Below you see an example of input specification for the Fusion Policy Learner:

1 <SieveFPL>
2	<Parameters>
3		<!--SelectionMethod name="MinAbsError"/-->
4		<SelectionMethod name="MaxCorrectValues" error="0.05"/>
5	</Parameters>
6	<Input>
7		<GoldStandard>gold\cities1000_gold_triples.nt</GoldStandard>
8		<dumpLocation>dumps-3cities</dumpLocation>	
9		<SieveExec>c:\ldif-0.5.1\bin\ldif.bat</SieveExec>
10	</Input>
11	<Output>
12		<SieveSpec>sieve-optimal\sieve_optimal.xml</SieveSpec>		
13		<FPLReport>FPL_report.txt</FPLReport>
14		<!--FPLReport valmatrix = "true">FPL_report.txt</FPLReport-->
15	</Output>
16	<Sieve xmlns="http://www4.wiwiss.fu-berlin.de/ldif/">
17		<Prefixes> 
18			<Prefix id="dbpedia-owl" namespace="http://dbpedia.org/ontology/"/>
19			<Prefix id="ldif" namespace="http://www4.wiwiss.fu-berlin.de/ldif/"/>
20			<Prefix id="sieve" namespace="http://sieve.wbsg.de/vocab/"/>
21			<Prefix id="dbpedia-meta" namespace="http://dbpedia.org/metadata/"/>
22		</Prefixes>
23		<QualityAssessment>
24			<AssessmentMetric id="sieve:authactivity">
25				<ScoringFunction class="NormalizedCount">
26					<Param name="maxCount" value="4250000"/>
27					<Input path="?GRAPH/dbpedia-meta:autheditcnt"/>
28				</ScoringFunction>
29			<AssessmentMetric id="sieve:recency">
30				<ScoringFunction class="TimeCloseness">
31					<Param name="timeSpan" value="500"/>
32					<Input path="?GRAPH/dbpedia-meta:lastedit"/>
33				</ScoringFunction>
34			</AssessmentMetric>
35		</QualityAssessment>
36		<Fusion>
37			<Class name="dbpedia-owl:PopulatedPlace">
38				<Property name="dbpedia-owl:areaTotal">
39					<FusionFunction class="KeepFirst" metric="sieve:recency"/>
40					<FusionFunction class="KeepFirst" metric="sieve:authactivity"/>
41					<FusionFunction class="Voting"/>
42					<FusionFunction class="Average"/>
43				</Property>
44				<Property name="dbpedia-owl:populationTotal">
45					<FusionFunction class="KeepFirst" metric="sieve:recency"/>
46					<FusionFunction class="Average"/>
47					<FusionFunction class="Maximum"/>
48				</Property>
49			</Class>
50		</Fusion>
51	</Sieve>
52 </SieveFPL>

The root tag is <SieveFPL>, under which four elements are to be specified: parameters of the learning algorithm (lines 2-5), input (lines 7-11) and output (lines 12-15) paths, and the extended Sieve specification (lines 16-20).

The <SelectionMethod> element is used to specify the parameters of the learning algorithm for numeric data values. Specifically, MinAbsError value of the name attribute corresponds to the 1st learning strategy introduced aboove, and the MaxCorrectValues – to the 2nd one with error attribute defining the maximum error threshold.

In the input section, the following parameters are specified:

  • <GoldStandard>: path to the gold standard dump, should be in N-TRIPLE format;
  • <dumpLocation>: path to the input dumps, dumps should be in N-QUAD format (the same requirement as for the input data in Sieve Data Fusion module);
  • <SieveExec>: path to the Sieve shell script or batch file.

The output section contains the following two paths to the output files generated by the tool:

  • <SieveSpec>: path to the resulting Sieve specification file;
  • <FPLReport>: path to the report summarizing the learning process.

Finally, <Sieve> element contains the extended or "redundant" Sieve specification: for each property a list of fusion functions is defined by the user, and the learner selects an optimal one from the list with respect to the gold standard.

In lines 39-42 four fusion functions (keep the most recent value, keep the value added by the most active author, most frequent, average) are specified for the areaTotal property of a populated place. The learner will choose and put into the final Sieve specification (line 12) only one of these three functions, in accordance with the selection method defined in <Parameters> element. Note that in the current version of the Fusion Policy Learner the learning can be performed for only one class at a time, that is learning the optimal fusion strategies for the properties of e.g. dbpedia-owl:CelestialBody would require another specification file.

3. Examples

The multilingual DBpedia example for the Fusion Policy Learner in the LDIF repository. The goal in this example is to fuse data for the same city from multiple language editions of DBpedia (and hence, Wikipedia). In the repository you find the input specification along with data dumps, provenance metadata and gold standard.
